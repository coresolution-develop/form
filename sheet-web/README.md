# 근무표 (시트 직결) — 서버 없는 단일 페이지 앱

`ex/`의 NestJS+MySQL+Redis 백엔드 **없이**, 브라우저가 Google 시트를 **직접** 읽고 쓰는 정적 웹앱입니다.
구성 요소는 딱 둘 — **Google 시트(데이터)** + **이 웹 페이지(UI)**. 서버·DB·env 없음.

- **인증**: Google Identity Services(GIS) OAuth 토큰 — 사용자가 브라우저에서 Google 로그인
- **데이터**: Google Sheets API v4 직접 호출 (`Bearer` 토큰, API 키 불필요)
- **집계**: 합계(M/HD/OFF/Y 등)는 `설정` 탭 규칙으로 **클라이언트에서 계산**
- **단일 진실원천**: 시트. 여러 조직 = 여러 시트(앱에서 시트 전환)

> 기존 `ex/` 시트 계약과 **호환**됩니다. 이미 쓰던 시트를 그대로 가리키면 됩니다.

---

## 1. Google Cloud 설정 (1회, 약 5분)

서버가 없으니 "설정"은 Google Cloud 쪽 OAuth 클라이언트 발급뿐입니다.

1. [Google Cloud Console](https://console.cloud.google.com/) → 프로젝트 생성/선택
2. **API 및 서비스 → 라이브러리 → "Google Sheets API" 사용 설정**
3. **OAuth 동의 화면** 구성
   - 사용자 유형: Workspace면 *내부(Internal)*, 아니면 *외부(External)*
   - 외부면 **테스트 사용자**에 본인(및 사용할 사람) Google 계정 추가
   - 스코프는 추가 안 해도 됨(앱이 런타임에 요청). 필요한 스코프: `.../auth/spreadsheets`
4. **사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
   - 애플리케이션 유형: **웹 애플리케이션**
   - **승인된 자바스크립트 원본(Authorized JavaScript origins)** 에 앱을 서빙할 origin 추가
     - 로컬: `http://localhost:8765` (아래 2번에서 쓰는 포트와 일치)
     - 배포: 예 `https://your-host.example.com`
   - 리디렉션 URI는 **불필요**(토큰 모델)
5. 생성된 **클라이언트 ID**(`...apps.googleusercontent.com`) 복사

> 클라이언트 ID는 비밀이 아닙니다(웹 OAuth는 *승인된 origin*으로 보호). API 키·시크릿 없음.

---

## 2. 실행

OAuth는 **승인된 origin에서 서빙된 페이지**에서만 동작합니다(`file://` 불가).

```bash
cd sheet-web
python3 -m http.server 8765      # 또는 임의 정적 서버
# 브라우저에서 http://localhost:8765 열기 (1-4의 승인된 origin과 동일해야 함)
```

첫 화면에서:
1. **OAuth 클라이언트 ID** 붙여넣기
2. **스프레드시트 ID 또는 URL** 붙여넣기 (`https://docs.google.com/spreadsheets/d/<ID>/edit` 통째로도 OK)
3. **Google 로그인 + 연결** → 동의 → 끝

입력값은 이 브라우저 `localStorage`에만 저장됩니다. 상단 **⚙ 시트** 버튼으로 다른 시트로 전환(= 다른 조직).

## 3. 시트 계약 (기존 `ex/`와 동일)

**월 탭** `{접두사}-YYYY-MM` (예: `근무표-2026-06`)

| | A | B | C | D … (1..N일) | 그 뒤 (버킷 합계) |
|---|---|---|---|---|---|
| 1행 | `YYYY-MM`(앵커) | | | | |
| 2행(헤더) | empId | 성명 | 직급 | 1 … N | M HD / Y … |
| 3행~ | empId | 이름 | 직급 | 근무코드 | 합계(자동) |

**`설정` 탭** (근무형태 사전)

| A | B | C | D | E … (버킷별 가중치) |
|---|---|---|---|---|
| code | label | bg(hex) | fg(hex) | M HD / Y … |
| `M` | 평일근무 | transparent | | 1 |
| `M/` | 오후반차 | #F1EFE8 | #5F5E5A | 0.5 … |

- **접두사·월 목록·`설정` 탭**은 탭 이름에서 **자동 감지**합니다(설정 불필요).
- 합계 = 각 칸 코드의 버킷 가중치 합 (반차 `M/` → 근무 0.5 + 연차 0.5 처럼 분할 기여 가능).

## 4. 할 수 있는 것 / v1 한계

**가능**: 월 그리드 보기·셀 편집(클릭 팝오버) → 시트에 즉시 기록 + 합계 자동 갱신 · 월 이동 · `설정` 탭 편집/저장 · 직원 추가/수정/행삭제 · **다음 달 준비**(마지막 존재 월의 다음 달 탭 생성 + 직원 명부 이월 + **기본 근무 자동 입력: 월~금 `M`, 토·일 `/`**) · **빈 시트 초기화**(`설정` 탭 기본 근무형태 25종 시드 + 이번 달 탭 생성).

> 월 이동(◀▶)은 **존재하는 탭만 읽습니다** — 탭이 없는 달은 "탭 없음"으로 표시되고 자동 생성하지 않습니다. 다음 달을 만들려면 **＋ 다음 달 준비**를 누르세요(월~금 M · 토·일 / 가 자동으로 채워집니다). 직원을 새로 추가할 때도 같은 기본 근무가 채워집니다.
> 빈 시트(설정/월 탭 없음)를 가리키면 상단에 **⚡ 시트 초기화** 버튼이 자동으로 나타납니다.

**아직 안 됨 (다음 단계 후보)**
- 동시 편집 충돌 제어 없음(마지막 쓰기 우선) — 실험 규모엔 충분.
- 토큰 만료(~1시간) 시 자동 재요청(필요하면 다시 동의).
- 셀 색/조건부 서식 등 시트 *서식*은 만들지 않음(값만 기록).

각 사용자는 대상 시트에 **본인 Google 계정으로 접근 권한**(보기/편집)이 있어야 합니다.
